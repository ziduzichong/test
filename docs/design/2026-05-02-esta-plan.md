## 执行顺序

```
任务1 (项目初始化)
  → 任务2 (数据库)
    → 任务3 (认证)
      → 任务4 (公告API) ─┐
      → 任务5 (成员API) ─┤  可并行
      → 任务6 (文件API) ─┤
      → 任务7 (账号API) ─┘
        → 任务8 (前端框架) 
          → 任务9 (公开页面)
            → 任务10 (开发者页面)
              → 任务11 (富文本+上传UI)
                → 任务12 (移动端适配)
```

**并行任务组**：任务 4、5、6、7 互不依赖，可同时编写。

## 关键决策点

| 决策点 | 说明 |
|--------|------|
| 富文本编辑器选择 | 使用 Quill.js CDN，无需 npm 安装，协会后续维护简单 |
| 文件存储路径 | `public/uploads/<category>/`，按分类分目录 |
| 初始管理员账号 | 首次运行时自动创建 admin/admin123，首次登录后建议修改 |
| 前端路由方案 | URL hash 路由（#/home 等），兼容性好无需服务端配置 |
| 密码安全 | bcryptjs 10 轮哈希，cookie httpOnly |
| 单文件架构 | server.js 包含所有路由，200行以内保持可维护性；如超出则按模块拆分 |
