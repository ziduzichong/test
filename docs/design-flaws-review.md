# 设计自审：未覆盖的隐藏问题

逐项审查当前架构设计，找出"在文档上看起来没问题，实际上会翻车"的点。

---

## 🔴 致命级：会导致网站不可用或数据丢失

### 1. 单人管理员的保障措施

**约束**：协会只有 1 个技术负责人，这是既定条件。不是设计缺陷，是需要配套保障。

**真实风险**：技术负责人毕业/失联 → 没人能进服务器。网站还在跑，但无法更新、备份、重启。

**保障措施**：
- Django superuser 密码纸质存档，密封在协会办公室（换届文档已更新）
- NAS 管理页面 / 服务器供应商有独立的重置入口（不依赖 SSH）
- 换届清单明确要求"交接服务器 root 密码 / NAS 管理密码"（开发文档第 11 节已更新）
- 运维手册覆盖全部日常操作，降低技术门槛

### 2. Django Migration 链断裂

**当前设计**：依赖 Django migration 机制保证数据库可追溯。

**真实场景**：2年换届4次，每次 `makemigrations` + `migrate`。第5次换届的新人发现 migration 文件有 30+ 个，不理解其中逻辑，或某次 migration 在本地能跑但在 NAS 上报错。新人可能：
- 删掉所有 migration 重新生成 → 数据库和数据丢失
- `migrate --fake` 跳过错误 → 表结构不一致 → 运行时崩溃

**修正**：
- 在开发文档中明确写：**永远不要删除 migration 文件，不要 `--fake` 除非你完全理解**
- 每次换届时执行 `python manage.py showmigrations` 确认全部 `[X]`
- 或者更激进的方案：**不用 migration，用 SQL 文件管理数据库版本**。对应每个表结构变动，手写 `sql/002_add_xxx_column.sql`，更直观但失去 ORM 的便利

### 3. Caddy 配置是隐形单点

**当前设计**：Caddy 负责 HTTPS + 文件直投 + 安全头 + 限流。配置写在一个 Caddyfile 里。

**真实场景**：Let's Encrypt 证书续期失败（域名过期、DNS 错误、Caddy 版本过旧），整个网站的 HTTPS 失效。新人面对 Caddy 报错不知道从何查起。更糟的是，如果 Caddy 挂了，Django 在 8000 端口还跑着，但外面完全访问不到。

**修正**：
- 在 `/health/` 端点返回 Django 状态，监控可以直接访问 8000 端口确认 Django 还活着
- 或者在 docker-compose 里加 `healthcheck`，Caddy 挂了至少 Docker 日志能看到
- 文档中补充：如何绕过 Caddy 临时访问 Django（`http://server:8000`）用于紧急排障

### 4. 批量导入中断无恢复

**当前设计**：`bulk_import` 命令一次性处理所有文件。如果 10,000 个文件中途崩溃（磁盘满、断电），已经导入的没有记录，没导入的也不知道从哪继续。

**真实场景**：导入到第 8,000 个文件时磁盘满了。命令崩溃。已经复制了 8,000 个文件到 `media/uploads/`，但数据库里只记录了 7,500 条（因为最后一批事务回滚了）。重新运行命令会发现有 7,500 个"重复"跳过，但有 500 个文件存在磁盘上却没有数据库记录。

**修正**：
- 改为分段事务（每 100 个文件 commit 一次），中断后可以续传
- 或者导入前先 `--dry-run` 统计，确认磁盘空间充足
- 命令末尾加一致性检查：`文件系统文件数 vs 数据库记录数`

## 🟠 严重级：功能缺失或体验显著降级

### 5. 10,000+ 文件没有搜索功能

**当前设计**：文件列表按分类筛选，翻页浏览。

**真实场景**：协会成员想找"上学期老师发的那个 STM32 定时器 PPT"。10,000 个文件，按分类筛选后还有 2,000 个，逐页翻找 → 放弃 → 去 QQ 群里问 → 网站白做了。

**修正**：
- 利用 SQLite FTS5 全文索引，给文件名和描述建索引
- 搜索框放在文件列表顶部，输入即搜

```sql
-- SQLite FTS5 全文搜索（不需要额外服务）
CREATE VIRTUAL TABLE files_fts USING fts5(
    original_name, description, content='files', content_rowid='id'
);
```

Django 端只需 `UploadedFile.objects.filter(original_name__icontains=query)` 简单实现，或用 FTS5 做更精准的搜索。无需 Elasticsearch。

### 6. 没有密码重置机制

**当前设计**：密码忘了 → 找管理员 → 管理员用 `changepassword` 命令重置。

**真实场景**：管理员自己忘了密码，或者管理员不在身边（暑假）。网站虽然没有停机，但没人能进去管理。

**修正**：
- 不需要 SMTP 邮件服务器。两种简单方案：

```python
# 方案A：安全问题重置（不依赖外部服务）
# 创建账号时设置 3 个安全问题，重置时回答

# 方案B：管理员物理接触服务器时用命令重置
python manage.py changepassword <username>
```

协会场景方案 B 够了（15 人，管理员总找得到）。但文档里要明确写出来——当前文档只提了 `changepassword admin`，没提普通用户也可以。

### 7. 校园内网无互联网的场景

**当前设计**：CKEditor CDN + Alpine.js CDN + HTMX CDN + Caddy Let's Encrypt。

**真实场景**：网站部署在校园内网的 NAS 上，服务器不能访问外网。pip install 失败（PyPI 不通）、CDN JS 加载不出、HTTPS 证书无法签发。

**修正**：
- JS 库全部本地化：下载 `alpine.min.js`、`htmx.min.js` 放到 `static/js/`，模板引用本地路径
- CKEditor 5 已经有 `django-ckeditor-5` 包，走本地静态文件
- 内网用 HTTP（不加 HTTPS），或使用内部 CA 签发的证书
- requirements.txt 可以用 `--find-links` 指向本地 wheel 包目录
- 在部署文档中增加"离线部署"一节

### 8. 重名文件覆盖问题

**当前设计**：`bulk_import` 检测到重名加后缀 `_1`, `_2`。网页上传用时间戳前缀 `1777726902118-filename.pdf`。

**真实场景**：老师更新了课件 V2，上传时文件名相同。网站存为不同的文件名（时间戳不同），但数据库里有两条 `original_name` 相同的记录。用户下载时看到两个同名文件，不知道哪个是新版。

**修正**：
- 数据库中加 `version` 字段，或 `updated_at` 区分
- 前端文件列表对同名文件加"最新"标记
- 或者更简单：允许覆盖。上传时如果检测到同名，弹窗问"替换旧版还是保留两份？"

## 🟡 中等级：不便但可容忍

### 9. Django Admin 对非技术人员不友好

**当前设计**：Django Admin 作为管理后台，"一行代码就有增删改查"。

**真实场景**：内容编辑（协会普通成员，不会编程）打开 Django Admin。全是英文标签。编辑成员时看到 `User` 和 `Member` 两个模型分不清。上传文件时不知道 `sha256_hash` 是什么。改公告用的 CKEditor 和 Admin 的样式冲突。

**修正**：
- Admin 中文化：`verbose_name` 全部写中文
- Admin 只给管理员用，普通编辑者的操作通过自定义管理面板（已有路由设计 `/dashboard/`）
- 实际上：Admin 给技术维护者用（1人），内容编辑用前端管理面板。这个分工在文档中要说清楚

### 10. 文件存储没有去重

**当前设计**：每个文件独立存储，按文件名+大小跳过重复。

**真实场景**：同一个 500MB 的 Keil 安装包被 3 个成员在不同时间上传了 3 次（文件名略微不同：`Keil_v5.exe`、`KEIL_V5.exe`、`keil-v5.exe`）。占用 1.5GB 存储，但本质上是同一个文件。

**修正**：
- 不是现在要做。小协会 50GB 够了，500MB 重复 3 次也不会满
- 但可以在设计文档中备注：未来如果需要，改 `bulk_import` 按 SHA256 去重

### 11. 大文件下载没有断点续传的服务端保障

**当前设计**：Caddy 支持 Range 请求（HTTP 标准），浏览器自动支持。

**潜在问题**：Caddy 默认确实支持 Range，但需要确认 `internal` 路由是否也支持。Caddy 的 `internal` 路由用于防止外部直接访问，可能对 Range 请求的处理有差异。

**验证方法**：部署后 `curl -H "Range: bytes=0-1023" http://server/files/1/download/` 确认返回 206 Partial Content。

## 🔵 低等级：不紧急但应记录

### 12. 没有监控告警

服务器挂了，没人知道，直到有人打不开网站。协会场景可以接受——但如果上课时挂了，老师等着用，就很尴尬。

简单方案：在 NAS 上设置一个定时任务，每 5 分钟 `curl http://localhost:8000/health/`，失败了发邮件/企业微信通知。

### 13. 没有文件预览

上传了 PDF、图片、代码文件，用户必须下载才能看内容。不能在线预览。影响体验但不影响功能。后续可加：PDF 用浏览器内置预览，代码用 highlight.js 渲染。

### 14. 视频没有缩略图和播放时间

文件列表只显示文件名和大小。视频文件不知道时长、没有封面。需要下载才知道是什么内容。这是"好用"层面的问题，不影响核心功能。

### 15. 换届后旧 migration 堆积

2年后有 30+ 个 migration 文件，每个新人看到都会紧张。解决办法：`squashmigrations` 把旧的压缩成一个，但操作有风险。

## 汇总：当前设计能做"小而精"吗？

| 维度 | 评价 | 说明 |
|------|------|------|
| 基础功能完整度 | ✅ 可以 | 公告、成员、奖项、文件管理都覆盖 |
| 安全基线 | ✅ 可以 | 防爆破、文件校验、下载鉴权已设计 |
| 300并发 | ✅ 可以 | 前提是 Caddy X-Accel-Redirect 配置正确 |
| 50GB初始导入 | ✅ 可以 | `bulk_import` 命令，但需加续传保护 |
| 换届移交 | ✅ 已加固 | migration 教程 + 运维手册 + 纸质密码存档 + 换届清单 |
| 非技术人员使用 | ⚠️ 部分 | Admin 英文 + 文件搜索缺失 |
| 内网离线部署 | ❌ 漏了 | CDN + PyPI 依赖假设外网可达 |
| 故障恢复 | ✅ 已加固 | 运维手册覆盖全故障场景 + 紧急恢复流程 |

> 结论：这个设计**可以做一个小而精的网站**，但前提是把上面 8 条"立即补"和"开发实现"的问题处理掉。最大的风险不在代码，在于**只有一个人能碰服务器**和**换届时的 migration 知识断层**——这两个是组织问题，不是技术问题。
